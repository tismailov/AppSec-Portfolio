# 002-NO_CAST.INTEGER_OVERFLOW

Тип задачи: Разметка сработки
Оценка: False Positive

Описание: возможность переполнения переменной типа int32_t при выполнении битового сдвига на 8 битов. 

Анализатор определил потенциальную проблемы при выполнении битового сдвига в переменной strength

```jsx
(strength << 8);
```

Анализ сработки:

1. В функцию tempCEFromIndexAndStrength в качестве аргумента передается переменная int32_t strength. 

```jsx
static inline int64_t tempCEFromIndexAndStrength(int32_t index, int32_t strength)
```

1. В самой функции выполняется битовый сдвиг переменой strength

```jsx
(strength << 8);
```

1. Согласно комментарию разработчика, значение strength должно помещаться в 2 бита (0b00..0b11). 

```jsx
// The strength must fit into 2 bits (UCOL_PRIMARY..UCOL_QUATERNARY).
```

1. При битовом сдвиге влево на 8 позиций эти 2 бита займут позиции 8-9 в 32-битном int32_t. Значащие биты не достигают знакового бита (позиция 31), следовательно переполнение невозможно.

Вердикт: False Positive
